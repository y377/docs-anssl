WHOIS lookup shows registrar, creation time, expiry time, registrant, DNSSEC, name servers, and EPP status codes for a domain. Open it at [WHOIS lookup](https://anssl.cn/console/tools/whois).

## Page layout

The WHOIS tool has two tabs:

- **Domain lookup**: run queries, view results, share, add monitoring, and create reminders.
- **Reminder list**: manage domain expiry reminders you created.

When opened from monitor details, WHOIS automatically receives the current domain and runs the query.

## Run a lookup

<Steps>

1. **Enter a domain**

   Type a domain such as `example.com`.

2. **Click Search**

   The tool requests WHOIS data and displays structured results.

3. **Review the result**

   Results include registrar, creation time, expiry time, registrant, DNSSEC, name servers, and domain status.

</Steps>

Query history is stored in the browser with up to 100 records. Use **History** to revisit recent domains, delete one item, or clear all history.

## Query results

### Basic information

The basic information card shows:

- Registrar.
- Creation time.
- Expiry time and remaining days.
- Registrant.
- DNSSEC status.

Expired domains show an expired tag. Active domains show remaining days.

### Name servers

The name server card lists NS records and provides a copy-all action.

### Domain status

The status card shows EPP status codes and translates known codes, for example:

- `clientTransferProhibited`: transfer prohibited.
- `clientUpdateProhibited`: update prohibited.
- `clientDeleteProhibited`: delete prohibited.
- `clientHold`: DNS service suspended.
- `pendingDelete`: pending deletion.
- `redemptionPeriod`: redemption period.

<Aside type="caution">
  If the status indicates hold, pending deletion, or redemption, check the domain in your registrar account as soon as possible.
</Aside>

## Quick actions

After a successful query, the result card provides:

- **Reminder**: create an expiry reminder for active domains.
- **Add monitor**: open [Domain monitoring](../monitor) with the current domain prefilled.
- **Share**: open the share dialog for the query result.

## Reminders

Reminders help track domain expiry dates.

<Steps>

1. **Run a WHOIS lookup**

   Confirm the domain is not expired and the result includes an expiry time.

2. **Click Reminder**

   Open the reminder dialog.

3. **Set days before expiry**

   Choose how many days before expiry to remind. The default is 30 days.

4. **Save**

   Saved reminders appear in the **Reminder list** tab.

</Steps>

In the reminder list, you can edit reminder days, delete reminders, and refresh the list.

<Aside type="note">
  Reminders are an aid for domain asset management. Actual renewal status is still determined by the registrar, and critical domains should use registrar auto-renewal.
</Aside>

## Use with other tools

- Add monitoring from WHOIS results to continuously track certificate status.
- Open WHOIS from monitor details to check expiry, registrar locks, and name servers.
- Before requesting certificates, use WHOIS to confirm the domain is still registered and active.

## FAQ

### Why are registrant details missing?

GDPR, registrar privacy protection, and TLD policy often hide detailed registrant data. This is normal.

### Is query history synced to my account?

No. Query history is stored in the current browser and is not synced across devices.

### WHOIS says the domain is active, but the website is down. What next?

Check name servers, DNS resolution, hold statuses, web server reachability, and certificate status.
